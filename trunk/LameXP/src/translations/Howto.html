Howto: Translate LameXP

If you are willing to contribute a new LameXP translation or update/fix an existing translation, then this guide will help you to do so :-)


The first step:

The latest versions of all existing LameXP language files can always be found on my public SVN server:
https://opensvn.csie.org/traccgi/mulder1984/browser/LameXP/src/translations

I highly recommend to base new translations on the English translation (LameXP-EN.loc), because I cannot guarantee the correctness of contributed translations.
Note: Please don't apply updates to your own working copy. Before you do any updates, grab the latest version of the language file from the SVN server!

(If you encounter any "security" errors when accessing the SVN server, you must import the CSIE root certificate into your browser first)


Editing the language file:

You can edit the language file with your favorite text editor. Anyway, I highly recommend the Notepad++ editor:
http://notepad-plus.sourceforge.net/

(Hint: You should set the "Language" to "MS INI" in Notepad++ for proper syntax highlighting!)


Language file editing rules:

Only language files that have been edited according to the following rules can be accepted:
  • Don't remove any lines and don't add any new lines. The number of lines must be identical before and after your editing.
  • Don't change the order of the lines. Keep the sorting as-is.
  • For each line: Don't change the text left of the equals sign (=), only translate the text right of it.
  • Take care of the C printf syntax: All the %s and %d placeholders must remain in the translated strings.
  • Don't touch the lines in square brackets (section headers) and don't remove the whitespace lines.

The proper Codepage:

Since LameXP does not support Unicode yet, we must use the proper Codepage for each language. The Codepage is specified in the corresponding line:

  • Codepage=1234

You must specify the correct Codepage here in order to make sure that your translation will be displayed properly on the Client's system.

A list of locales and the corresponding ANSI Codepages can be found in this article:
http://msdn.microsoft.com/en-us/library/ms776260(VS.85).aspx

If you still don't know what your Codepage is, simply use my tool here:
https://opensvn.csie.org/traccgi/mulder1984/export/head/LameXP/src/translations/WhatsMyCodepage.exe

Important: Please note that in the language file we only need the decimal Codepage number, not the full codepage name or description!


Personalization:

You can put the name of your language and your own name (will be displayed in the application) into the corresponding lines:

  • Language=English
  • Contributor=John Doe

Testing your translation:

Since I probably don't understand your language, I can't check or correct your translation. So it's important that you test your language file thoroughly!

You can simply put your new language file (*.loc) into the same directory as 'LameXP.exe' and it will appear in LameXP's language menu upon next launch.
This way you can test your language file easily before submitting it. Only tested language files can be included in official LameXP releases.

When testing a translation the proper Codepage must be used. You can quickly switch the Codepage for LameXP with the AppLocale utility:
http://www.microsoft.com/globaldev/tools/apploc.mspx

Important: Please make sure that your translation does not destroy LameXP's layout! You can feel free to use a shorter (but meaningful) text, if required.


Submitting your translation:

Once you are done with your translation, you can submit it, so it can be included in the next LameXP release. You can contact me by e-mail:
mulder2[at]gmx[dot]de

It is sufficient to submit the language file (*.loc). You don't need to compress the language file (ZIP, RAR, etc) and you don't need to attach a flag icon file.
Furthermore your language file should use the LameXP-XX.loc naming scheme, where XX is the corresponding upper-case ISO 639-1 language code.

(Note: Advanced translators can get write-access to the SVN repository on request, so they can keep their translations up-to-date easily)



Thanks in advance !!!
